# Administrar complementos

Partes de este tema pueden estar traducidos automáticamente.

# TrueConf Directory

En la sección Extensions → TrueConf Directory puede configurar la integración de la instancia de TrueConf Server, parte de la solución integral TrueConf Enterprise, con la solución TrueConf Directory.

Para hacerlo, haga clic en el botón Activate. Para desactivar la integración, haga clic en el botón Deactivate.

En el campo grande debajo de la tabla, aparecerá la clave secreta generada.

Para obtener más información sobre la configuración de la extensión TrueConf Directory y su adquisición, póngase en contacto con nosotros de la manera que le resulte más conveniente.

Las capacidades de TrueConf Directory las demostramos en ISE 2019.

# Integración con DLP

Si su licencia de TrueConf Server tiene activada la extensión Integration with DLP, entonces en esta sección podrá configurar la conexión al sistema y las acciones en caso de detección de intentos de violación de las normas de seguridad.

  1. Activación de la extensión. Mientras la casilla no esté marcada y los ajustes no se guarden utilizando el botón Apply, ninguna verificación funciona.

  2. Parámetros de conexión al sistema DLP: host (IP o FQDN sin el prefijo http: / https:), puerto y tipo de conexión (normal o protegido por TLS).

  3. Comprobación de disponibilidad del sistema. Verá el resultado de la comprobación en la línea de estado a continuación.

  4. Activación de la verificación de mensajes de texto enviados. Mientras la casilla no esté marcada y la configuración no se haya guardado utilizando el botón Apply, la verificación no funciona.

  5. Solicitud ICAP que contiene los campos necesarios para transmitir al sistema DLP. El formato de la solicitud depende del sistema en particular, a continuación se enumeran las variables utilizadas en la plantilla que se reemplazan con valores específicos al analizar los datos.

  6. Restablecer la solicitud de ICAP al patrón predeterminado.

  7. Cancelación en la solicitud de los últimos cambios que no se han guardado utilizando el botón Apply.

  8. Acciones con el mensaje que se ha reconocido como no deseado después del análisis por parte del DLP. Puede reemplazar el mensaje con el texto seleccionado en el sistema DLP, especificar su propia versión, o dejarlo sin cambios. En este último caso, los usuarios recibirán todos los mensajes, pero en los registros del sistema DLP se registrará el envío de mensajes no deseados.

  9. Acción con un mensaje no deseado en caso de no haber conexión con el sistema DLP. Por ejemplo, se puede especificar el texto "No hay conexión con el sistema de seguridad" para que ningún mensaje pueda llegar a los destinatarios hasta que se resuelva el problema.

  10. Tiempo de espera para la conexión con el sistema DLP. Si se pierde la conexión con el sistema, el tiempo especificado aquí pasará cuando se envíe el primer mensaje después de este evento antes de que TrueConf Server responda de acuerdo con la configuración del punto 9 (ya que se intentará restablecer la conexión). Posteriormente, la verificación de la conexión se realiza en segundo plano, y la transmisión de mensajes o su prohibición ocurrirá prácticamente al instante.

  11. Activación de la verificación de archivos enviados. Mientras no se marque la casilla y no se guarden los ajustes con el botón Apply, la verificación no funciona. Cabe señalar que, dependiendo del tamaño del archivo, puede llevar algún tiempo analizarlo y obtener una respuesta del sistema, ya que primero se carga en TrueConf Server, y luego al sistema DLP.

  12. ICAP-solicitud que contiene los campos necesarios para transferir al sistema DLP.

  13. Restablecer la solicitud de ICAP al patrón predeterminado.

  14. Cancelación en la solicitud de los últimos cambios que no se han guardado utilizando el botón Apply.

  15. Acciones con el archivo que se considera indeseable tras el análisis por parte del DLP. Es posible reemplazar el archivo con un texto seleccionado en el sistema DLP, especificar su propia versión, o dejarlo sin cambios. En este último caso, los usuarios recibirán todos los archivos, pero en los registros del sistema DLP, se registrará el envío de archivos indeseables.

  16. Acción con un archivo no deseado en caso de no haber conexión con el sistema DLP. Por ejemplo, se puede indicar el texto "No hay conexión con el sistema de seguridad" para que ningún mensaje pueda llegar a los destinatarios hasta que se resuelva el problema.

  17. Tiempo de espera para la conexión con el sistema DLP. Se utiliza la misma lógica de funcionamiento que para los mensajes, descrita en el apartado 10.

  18. Limitación del número de participantes en el chat grupal para enviar su lista a DLP.

  19. No olvide hacer clic en el botón Apply para guardar los cambios.

# Variables en las plantillas de solicitudes ICAP

  • %body — contenido de la solicitud (mensaje de texto del chat)

  • %body_length — longitud del contenido de la solicitud (en bytes)

  • %body_offset — desplazamiento del contenido de la solicitud en la sección encapsulada (en bytes)

  • %chat_id — identificador único GUID del chat

  • %chat_id_origin — %chat_id desde donde se reenvía el mensaje (si no se reenvía, queda vacío)

  • %chat_title — nombre del chat

  • %chat_title_base64 — %chat_title en formato base64

  • %content_length — la longitud del contenido de la solicitud (decimal, en bytes)

  • %date — fecha en formato ISO 8601 (opens new window)

  • %dst — el TrueConf ID completo del destinatario en forma de user@server

  • %dst_base64 — %dst en formato base64 (opens new window)

  • %dst_user — es el nombre de usuario del destinatario (parte de TrueConf ID antes del símbolo @) con el dominio especificado como domain\user

  • %dst_user_at_domain — el inicio de sesión del destinatario en formato user@domain (puede ser sin @domain si está en el dominio principal)

  • %dst_user_at_domain_base64 — %dst_user_at_domain en formato base64

  • %dst_user_base64 — %dst_user en formato base64

  • %dst_user_no_domain — login del destinatario

  • %dst_user_no_domain_base64 — %dst_user_no_domain en formato base64

  • %host — el valor se toma del campo Host

  • %port — el valor se toma del campo Port

  • %server_name — nombre de dominio de TrueConf Server

  • %src — el TrueConf ID completo del remitente

  • %src_base64 — %src en base64

  • %src_user — es el nombre de usuario del remitente (parte del TrueConf ID antes del símbolo @) con el dominio incluido de la forma dominio\usuario

  • %src_user_at_domain — el nombre de usuario del remitente en formato user@domain (puede estar sin @domain si está en el dominio principal)

  • %src_user_at_domain_base64 — %src_user_at_domain en formato base64

  • %src_user_base64 — %src_user en formato base64

  • %src_user_no_domain — login del remitente

  • %src_user_no_domain_base64 — %src_user_no_domain en formato base64

  • %src_ip — dirección IP del remitente

Para los archivos, además están disponibles:

  • %filename — nombre del archivo enviado

  • %filename_base64 — %filename en formato base64

# Complementos de correo

Esta extensión permite:

  • administrar la versión web del complemento para Microsoft Outlook, que se descargará directamente desde su servidor;

  • obtener enlaces directos para instalar la versión para Windows del complemento para Outlook y ;

  • configurar la plantilla de invitación a la reunión.

Esta extensión se proporciona de forma gratuita, incluida con la versión TrueConf Server Free.

En la pestaña Plugins podrá:

  1. En el bloque Outlook (web) — descargar el archivo xml para instalar la versión web de la extensión (plugin), así como actualizar la versión actual en el servidor. El enlace para instalar el plugin se puede copiar usando el botón y distribuir entre los usuarios de la red corporativa (incluida la cerrada sin acceso a Internet), para que puedan instalarlo directamente desde su TrueConf Server.

  2. En el bloque Outlook (for Windows) — copie el enlace para instalar la versión de escritorio del complemento para Outlook utilizando el botón y distribúyalo entre los usuarios. Ellos podrán instalar el plugin por sí mismos a través de Internet desde nuestro sitio web. Además, podrá distribuir la aplicación de forma centralizada utilizando políticas de grupo, ya que se proporciona en forma de paquete msi.

Obtenga más información sobre la instalación y las funciones de las versiones de escritorio y web de la extensión para MS Outlook en nuestra base de conocimientos.

3. En el bloque Thunderbird — copie el enlace para instalar el plugin para la aplicación Thunderbird usando el botón y distribúyalo entre los usuarios.

En la pestaña Settings podrá modificar el texto de la plantilla de descripción que se añade al crear un evento a través de cualquiera de los plugins de correo de TrueConf. También puede activar aquí la inclusión del código PIN para la conexión (si está disponible) en la descripción:

En la plantilla de invitación se pueden utilizar una serie de constantes, de manera similar a las plantillas de correo electrónico en la configuración de SMTP:

  • %conf_name — el nombre de la conferencia;

  • %conf_id — ID de la conferencia (por ejemplo, \c\df0a2adebe);

  • %owner_name — el nombre mostrado del propietario de la reunión;

  • %conf_url — enlace a la página de la conferencia, por ejemplo:

    https://example.com/c/CID

  • Parámetros para especificar los contactos del administrador del servidor:

    • %admin_name — nombre mostrado;

    • %admin_email — e-mail para contacto;

    • %admin_phone — teléfono de contacto.
